# Screenshot Acceptance Update - Complete Summary

## Overview
Updated the system to accept **screenshots of civic locations** while rejecting ONLY **AI-generated images**.

### Flow
```
User Uploads Image
    ↓
[Python Analyzer - Early Screenshot Detection]
    ├─ Is it a screenshot? → ACCEPT ✅ (return isCivicIssue: true)
    └─ Not a screenshot? → Continue to civic issue detection
        ↓
    [Civic Issue Detection]
        ├─ Shows civic issue (garbage, pothole, etc.)? → ACCEPT ✅
        └─ Doesn't show civic issue? → REJECT ❌
    
[Backend Image Verification - Sightengine Layer]
    ├─ Is it AI-generated (>60% confidence)? → REJECT ❌
    └─ Not AI-generated? → ACCEPT ✅
```

## Image Verification Strategy (3 Layers)

### Layer 1: Python AI Service (analyze_service)
| Image Type | Detection | Result |
|---|---|---|
| **Screenshot of map** | Screenshot detection (35%+ confidence) | ✅ ACCEPT |
| **Real photo of garbage** | Civic issue detection | ✅ ACCEPT |
| **Real photo of pothole** | Civic issue detection | ✅ ACCEPT |
| **Screenshot of random place** | Not civic-related | ❌ REJECT |
| **Photo of car** | Not civic-related | ❌ REJECT |
| **AI-generated civic scene** | Civic issue detection (passes) | ⚠️ PASS to Layer 2 |

### Layer 2: Sightengine API (imageVerificationService)
| Image Type | Detection | Result |
|---|---|---|
| **Screenshot (real)** | Normal image type | ✅ ACCEPT |
| **Real photo** | Real photo type | ✅ ACCEPT |
| **AI-generated image** | AI confidence > 60% | ❌ REJECT |
| **Animated image** | Animated detection > 50% | ❌ REJECT |

### Layer 3: Image Sanity Check
- File size: 20KB - 15MB
- Format: JPG, PNG, WebP only
- Resolution: minimum 400x400 pixels

## Use Cases Now Supported

### ✅ Citizen uploads location screenshot
1. Takes screenshot of map/location showing civic area
2. Python analyzer detects screenshot → ACCEPTS
3. Sightengine checks if AI-generated → REJECTS only if AI
4. Results in UI: "✅ Screenshot accepted! Add description to continue."

### ✅ Citizen uploads real photo of issue
1. Takes photo of garbage dump/pothole
2. Python analyzer detects civic issue → ACCEPTS
3. Sightengine checks authenticity → ACCEPTS real photo
4. Results in UI: "✅ Image verified! Valid civic issue detected."

### ✅ Citizen uploads AI image of issue
1. Uploads screenshot or image generated by AI
2. Python analyzer may accept (if detects civic pattern)
3. Sightengine detects AI (>60%) → REJECTS
4. Results in UI: "🤖 AI generated images are not accepted as evidence."

### ❌ Citizen uploads irrelevant screenshot
1. Takes screenshot of random website/app
2. Python analyzer doesn't detect screenshot (confidence < 35%) AND not civic issue → REJECTS
3. Results in UI: "Upload a real photo or location screenshot of the civic area"

## Python Code Logic

```python
@app.post("/analyze")
async def analyze(file):
    image = load_image(file)
    
    # Step 1: Check if screenshot
    screenshot_confidence = max(clip_probs(image, SCREENSHOT_PROMPTS))
    if screenshot_confidence > 0.35:
        return {
            "isCivicIssue": true,
            "isScreenshot": true,
            "category": "Location Evidence",
            ...
        }
    
    # Step 2: Check if civic issue
    target_confidence = max(clip_probs(image, TARGET_CLASS_PROMPTS))
    non_target_confidence = max(clip_probs(image, NON_TARGET_PROMPTS))
    
    if target_confidence >= 0.33 and (target_confidence - non_target_confidence) >= 0.08:
        return {
            "isCivicIssue": true,
            "category": detected_category,
            ...
        }
    
    # Step 3: Reject
    return {
        "isCivicIssue": false,
        "reason": "Not a civic issue or screenshot"
    }
```
